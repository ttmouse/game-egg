## Canvas 尺寸适配修复 (720x576)

### 问题
画面下半部分显示黑色，地面没有填充整个视口。

### 根本原因
原始设计为 720x720 canvas，地面和天空渐变坐标硬编码。改为 720x576 后：
- `canvas.height = 720` → `576` 但其他坐标未同步调整
- 等轴测地面 OY=100 时只占 canvas 的 24-45%
- 天空渐变和山也未按比例缩放

### 修复内容

1. **Canvas 尺寸** (line 438)
   - `canvas.height = 576`

2. **等轴测地面原点 OY** (line 636)
   - `OY = 100` → `OY = 300`
   - 让地面下降填充 canvas 下半部分
   - 验证：草地底部 y=482，占 canvas 的 84%

3. **天空渐变** (drawBg, drawBgToCtx)
   - 白天：5层渐变 (0-60, 60-120, 120-180, 180-240, 240-300)
   - 夜晚：5层渐变 (同结构，深色调)

4. **背景山** (drawBg, drawBgToCtx)
   - `mtnBase = 195` → `mtnBase = 300`
   - 山峰位置同步上移，与 OY=300 的 iso 网格对齐
   - 确保山在 iso 网格后面（正确的深度顺序）

5. **地面纹理坐标** (texDetails)
   - y 值增加 +200（从 OY=100 到 OY=300 的总偏移）

### 修复后布局
- 天空渐变：y=0-300 (52%)
- 背景山：y=220-300
- 等轴测网格：y=300-460 (52-80%)
- 草地：y=402-482 (70-84%)
- 空白：y=482-576 (9.4%)

### 关键代码位置
- Canvas: line 438
- OY 常量: line 636
- 天空渐变: drawBg (line ~1495), drawBgToCtx (line ~3411)
- 背景山: drawBg (line ~1508), drawBgToCtx (line ~3433)
- 地面纹理: texDetails 数组 (line ~707)
